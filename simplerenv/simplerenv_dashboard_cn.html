Jie Wang · everloom-129.github.io · GRASP Lab, UPenn  ·  Co-authored with Claude Code  ·  2026.6.1

SimplerEnv 通过缩小与两套真实平台的视觉差距，在仿真中评估真实机器人操作策略 — Google 的 RT-1 机器人 与 WidowX + Bridge 机械臂 — 从而让仿真成功率能够预测真实成功率，而无需运行实机硬件。两种评估模式：Visual Matching（叠加真实背景、匹配纹理）与 Variant Aggregation（扫描光照 / 背景 / 干扰物）。本图谱呈现逐任务的成功率矩阵，并揭示了核心发现：能力无法在不同本体（embodiment）间迁移。

为何采用真实到仿真。 在真实机械臂上运行 VLA 进行数百次试验既缓慢又难以复现。SimplerEnv 为两个真实实验室构建数字孪生（digital twin），使策略的仿真成功率能够追踪其真实成功率。问题在于：针对某一本体的动作空间与相机调优的策略，在 Google Robot 上可能接近 SOTA，而在 WidowX 上却几乎为零。下方数字均为成功率 %；WidowX 数据取任务成功（task-success）列（另有独立的抓取（grasp）子指标）。Place in Closed Drawer（指令：“place apple in closed top drawer”）已列入任务目录，但遵循 HPT/TraceVLA/SpatialVLA 的做法从结果矩阵中省略，因为几乎所有策略在该任务上的得分都约为 0。

      本体差异

      Google Robot vs WidowX · 同一策略，两种本体
      每条线 = 一个策略 · 左点 = Google Robot（VM 均值）· 右点 = WidowX（总体）· 线越长 = 迁移越差
      OpenVLA 是一个警示案例：在 Google Robot 上达到 27.7%，但在 WidowX 上仅 1.0% — 这是已知的控制器/动作空间问题（artifact），而非真实世界中的能力差距。即便是最强的策略 SpatialVLA，也从 71.9% → 34.4% 大幅下滑。综述得出的教训是：始终按本体分别报告。
