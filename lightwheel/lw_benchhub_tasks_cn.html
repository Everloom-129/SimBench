LW-BenchHub 是 Lightwheel 的统一评估枢纽——它将多个现有基准的任务整合进单一的评测框架与 gym 接口中。本次导出涵盖从 LIBERO 与 RoboCasa 抽取的 275 个任务，跨越 28 个任务套件（suites），每个任务都携带 VLA 所依据的原始语言指令，以及该指令是固定型还是模板型。

这是什么：LW-BenchHub 本身并不引入新的任务——它是一个集成层，将 LIBERO 与 RoboCasa 的任务重新托管在统一的 gym_id 方案下（注意包裹 LIBERO 任务的 Robocasa-Task-* 前缀）。这使得通过单一 API 在异构基准上运行同一策略变得十分便利，但底层的任务难度与成功指标仍归属于原始基准。

指令类型
模板型指令包含按场景（物体、位置）填充的槽位；固定型指令则是完全指定的字符串。两者的配比对评估语言接地（grounding）至关重要——模板型任务能探查策略是否真正读取了指令。

每条语言指令的词数——任务时间跨度的代理指标
较短的指令（约 8 词以下）大多是 LIBERO/RoboCasa 的原子化单阶段任务；长尾（20 词以上）则是 RoboCasa 的多阶段活动，在一句话中列出了多个子目标。
